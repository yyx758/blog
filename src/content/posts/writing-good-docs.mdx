好的文档是软件工程中最被低估的技能之一。以下是我总结的一些实用原则。

## 从"为什么"开始

写文档之前，先问自己：**这是给谁看的？他们想解决什么问题？**

一个以安装说明开头的 README 没问题。但一个以"这个库通过 X 方式解决 Y 问题"开头的 README 更好。

## 保持简短

开发者会扫读。请尊重他们的时间：

- 用标题拆分内容
- 把最重要的信息放前面
- 代码示例 > 长篇解释

## 展示，不要只说

```python
# 不好："使用 fetch 函数来获取数据"
# 好：
import asyncio
from mylib import fetch

async def main():
    data = await fetch("https://api.example.com/users")
    print(data)  # => [{"id": 1, "name": "Alice"}, ...]

asyncio.run(main())
```

一个能跑的示例胜过千言万语。

## 文档也是代码

把文档当代码来对待：

- 跟它描述的代码一起版本管理
- 在 PR 中一起 review
- 测试代码示例确保它们真的能跑

最好的文档之所以能保持更新，是因为它住在代码旁边。
